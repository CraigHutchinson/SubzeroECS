SubzeroECS Commercial License (Draft Summary)
=============================================

Request Process:
Open a new issue using the “Commercial License Request” template:
Title: Commercial License Request: <Your Organization Name>
Include:
- Organization / legal entity name
- Intended use (embedding, SaaS hosting, internal tooling)
- Whether modifications will be kept private
- Expected scale (users / deployments)
- Desired term (perpetual + maintenance, annual subscription)
- Any required clauses (indemnity, SLA, security review, audit exclusion)
- Target decision date

Do NOT provide sensitive, confidential information in the public issue; you can request moving to a private channel for detailed negotiation.

Scope (Proposed Terms Overview – not legally binding):
- Grant: Non-exclusive worldwide license to use, modify, embed, distribute in proprietary form.
- Copyleft Waiver: No obligation to release derivative source under AGPL.
- Source Confidentiality: Allowed to keep modifications private.
- Attribution: Optional “Powered by SubzeroECS” (can be waived).
- Patents: Limited patent license for any patents owned covering contributed code (final terms negotiable).
- Fees: Tiered (user count / revenue / distribution model).
- Support & SLA: Available as an add-on (define response times).
- Updates: Access to new versions during active term.
- Termination: For breach; post-termination cease distribution except for already shipped binaries (define survival clauses).
- Liability: Standard limitation of liability and disclaimer with optional negotiated caps.
- Security: Option for private security advisories before public disclosure.

Next Steps:
1. Open the issue template.
2. Provide required info.
3. Move to private negotiation (email/DocuSign/etc.) once initial scope confirmed.
4. Execute agreement; receive commercial license key/reference (if applicable).
