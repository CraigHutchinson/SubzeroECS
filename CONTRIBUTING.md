Contributing to SubzeroECS
==========================

License Model:
SubzeroECS is dual-licensed under:
- AGPL-3.0-only
- A proprietary commercial license (available via issue requests)

Inbound Contribution Policy:
By submitting a contribution you certify:
1. You have the rights to contribute it.
2. You grant Craig Hutchinson permission to distribute under AGPL-3.0 and to include it in commercial license offerings (dual licensing).

Mechanism:
Developer Certificate of Origin (DCO):
Add to each commit message:
Signed-off-by: Your Name <email@example.com>

(If a formal CLA is introduced later, instructions will be added.)

Commercial Requests:
Open an issue using the “Commercial License Request” template; do not email.

Code Standards:
- Keep changes focused.
- Add tests for new features.
- Maintain consistent style (add STYLE.md if needed).
- Avoid incompatible license dependencies.

Pull Request Checklist:
- Clear title & summary.
- Reference related issue if applicable.
- Signed-off-by line present.
- No proprietary third-party code.

Security:
Report vulnerabilities by opening a “Security Report” issue (or provide private channel later if established). Do not include exploit details publicly until coordinated disclosure is agreed.

 * See LICENSE for details.
 */

Questions:
Open a general issue; label appropriately (e.g., question, discussion).
